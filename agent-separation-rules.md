# Правила разграничения работы агентов: дашборд Cuteam и mini

## Кратко
- Codex: только новый модуль дашборда `src/features/cuteam/**` + точечное подключение в меню/роуты после D1.
- AG: только `/mini/**`.
- Общие файлы (layout, menu, routes, app shell, конфиги) — только по согласованию, отдельным коммитом.

## Зоны ответственности

### Codex (дашборд Cuteam)
Разрешено:
- `src/features/cuteam/**`
- `Показатели/**` — только чтение; правки по согласованию
- общие файлы (menu/routes/layout) — **только после D1** и по согласованию

### AG (mini)
Разрешено:
- `/mini/**`

Если mini пока размещен внутри `backend/app`, считать зоной AG:
- `backend/app/templates/mini.html`
- `backend/app/static/mini.js`
- функции и эндпоинты `/api/mini/*` в `backend/app/main.py` (только mini-зона)

## Git
- Ветки: `feature/cuteam-dashboard` (Codex), `feature/mini` (AG).
- Префиксы коммитов: опционально `[DASH]` и `[MINI]`.

## Протокол работы
1. Не работать одновременно в одной ветке.
2. Перед началом — `git pull`.
3. Общие файлы: согласовать, делать отдельный маленький коммит.
4. После `git push` — уведомить другого агента.
